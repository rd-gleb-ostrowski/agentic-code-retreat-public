# Menu Projects

The non-Arena **Track**: small, self-contained, whimsical (fantasy / sci-fi) projects a team
can start greenfield in the morning and optionally hand to another team in the afternoon.

Each project has a **participant handout** — a clear specification with explicit acceptance
criteria, golden example cases, and an **Escalation Roadmap** whose tiers each state their own
acceptance criterion, so no team ever runs out of work. Tests are described stack-neutrally:
turn the acceptance criteria and golden examples into tests in whatever language you choose.

## Facilitator only

[`CURVEBALLS.md`](./CURVEBALLS.md) holds the **Curveballs** — changing requirements the
facilitator drops in mid-Session to stress-test the architecture. **Do not hand this to
participants.** Throw a curveball when a team is comfortable; a well-architected codebase
absorbs it, a tangled one fights back — that contrast is the teaching moment for the next
mini-retro.
